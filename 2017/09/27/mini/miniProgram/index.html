mini program

小程序的特点

  • 基于包的管理
  • 用完即走，可以吸引不经常用你产品的客户变成你的忠实顾客

rpx

  • rpx（responsive pixel）
  • 1rpx=0.5px=1物理像素。Photoshop里面量出来的尺寸为物理像素点。所以可以直接使用标注尺寸数据

app.json的配置文件主要内容

  1. pages 页面名字路径数据
  2. window 窗口表现 设置小程序的状态栏、导航条、标题、窗口背景色
  3. tabBar 指定 tab 栏的表现，以及 tab 切换时显示的对应页面。
  4. networkTimeout 设置各种网络请求的超时时间
  5. debug

模板的使用

  • <include src="" />
  • <import src="" /> 结合<template is="模板名字" />

绑定事件

  • bind 会冒泡
  • catch 不会冒泡

app配置

  • tabBar 配置主义事项
    • list中的项，需要在pages里有的页面才会显示tabBar

App()里的函数

  • onLaunch 启动
  • onLoad 加载数据
  • onShow 绑定数据
  • onReady 渲染完成

导航有两种方式

  1. 用标签<navigator url=">
  2. api wx.navigateTo()，
    被跳转页面的onLoad: function (options) {} 可以接受查询字符串

布局

  • flex布局
  • 相对定位绝对定位

flex

  • 项目的属性 flex 是flex-grow flex-shrink flex-basis 三个属性的缩写，即放大倍数、缩小倍数、自身空间。

伪类选择器

  • 动态伪类选择器 只支持:active :focus

开发中遇到的问题

  1. 公众号账号和小程序账号不是一个啊
  2. 绝对定位absolute 是相对于最近的已经定位的父元素的
  3. 在做conde社区案例时候，导航这困扰我小一个小时，因为navigateTo 导航到了tabBar的页面，这是不行的。可以用switchTab()导航到tabBar的页面

心得

  1. 小程序中有vue和es6的影子
  2. wxml中可以用逻辑 这样布局代码就可以减少
  3. 基于腾讯云做后端接口 可以开发项目 试一试
  4. 微信开打时间有点长，有白屏，小程序可以解决，让用户体验更好；对于开发者也是降低成本的
